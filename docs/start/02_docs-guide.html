文档指南¶

本指南解释了 ROS 1 和 ROS 2 文档基础结构。它有助于理解具体资源在哪里，如何提问，以及维护哪些站点。

ROS Answers¶

目的

类似于 Stack Exchange 的 ROS 1 和 ROS2 的问答社区网站。

参考

如何使用 ROS Answers 请参见 联系 页面。

ROS Design¶

目的

一个记录许多早期 ROS 2 设计决策的地方。

说明

在 ROS 设计中，更可取的是 ROS 改善建议（REPs）。

ROS Discourse¶

目的

一个为 ROS 社区提供一般性讨论和公告的论坛。

参考

有关 ROS 讨论的更多信息，请参见 联系 页面。

ROS Docs¶

目的

  • ROS 1 和 ROS 2 API 文档

  • ROS2 核心文档，如安装说明、教程、指南、路线图等。

ROS Enhancement Proposals (REPs)¶

目的

一个 ROS1 和 ROS2 设计和约定的地方。

注解

代表比 ROS 设计 更可取，因为他们有一个更成熟的审查过程。

ROS Index¶

目的

链接到附加信息的所有包的索引列表。

注解

ROS 索引

  • 查看软件包支持哪些 ROS 发行版（ROS 1 和 ROS 2）

  • 链接到包存储库、API 文档或网站

  • 检查包的许可、构建类型、维护人员、状态和依赖项

  • 在 ROS Answers 中找到涉及包的问题

您可以将 ROS 2 索引看作类似于 ROS 包的 Python 包索引（PyPi）。

ROS Prerelease¶

目的

一个网站，帮助您生成命令来模拟本地机器上的 ROS Buildfarm。

注解

目前，这个网站的前端只显示ROS 1的分布。

ROS Robots¶

目的

一个展示使用 ROS 1 或 ROS 2 的机器人的地方。

注解

本页上的机器人由社区贡献，请参阅该网站了解如何贡献机器人。

ROS Wiki¶

目的

ROS1 文档和用户可修改的内容。

注解

  • 我们不会在 ROS 2 中使用 wiki，因为审核 wiki 的开销太大。ROS2 的 wiki 功能将被 ROS Docs (this site), ROS Index, ROS Discourse，和 ROS Answers 替换。

  • wiki 将至少在最后一个 ROS 1 发行版 EOL 之前激活。

ROS.org¶

ROS 1 和 ROS 2 产品登陆页面，提供了 ROS 的高级描述，并链接到其他 ROS 站点。

废弃¶

ROS 2 Docs

ROS 2 文档为包括 Galactic 在内的发行版提供 API 文档。从 Humble 开始，API 文档将托管在 ROS Docs 上。
